## Implementation Notes (2026-05-26)

### Database

New table `contract_generations` created via migration `20260526_contract_04f_generation_history.sql`.

Fields: id, student_id, application_id, generated_by, generated_at, file_name, storage_path, template_version, status, created_at.

Indexes on student_id, application_id, and generated_at desc.

RLS enabled:
- All authenticated staff (super_admin, admin, sales, viewer) can select.
- Only admin/super_admin can insert or update.
- No delete policy.

### Generation Behavior

When admin/super_admin generates a DOCX:
- DOCX is generated and immediately downloaded (existing behavior preserved).
- Generated DOCX is uploaded to Supabase Storage bucket `student-documents` at path `{studentId}/contracts/{timestamp}-{fileName}`.
- A `contract_generations` record is created with student_id, application_id, generated_by, generated_at, file_name, and storage_path.
- Application status updates to `contract_generated` if currently `ready_for_contract`.
- `contract_generated_at` is updated on every generation.
- `ready_for_contract_at` is not modified.
- If storage upload fails, the generation record is still created with null storage_path.
- If record creation fails, the DOCX download still succeeds. Error is logged server-side.

### Student File UI

The contract section (section 11) shows:
- Latest generated contract with file name, generated date/time, and generated by name.
- Download button for latest if storage_path exists.
- "Downloaded at generation time" note if no storage_path.
- Expandable history list for previous generations with download links.

### Role Behavior

- admin/super_admin: can generate DOCX, can see generation history, can download stored contracts.
- sales: can see generation history (read-only), cannot generate.
- viewer: can see generation history (read-only), cannot generate.

### Storage

Generated DOCX files are stored in the existing `student-documents` Supabase Storage bucket. The download reuses the existing `/api/documents/download` API route.

### Limitations

- Template version tracking is not populated in this ticket. The field exists for future use.
- If the Supabase Storage bucket does not exist in the target environment, storage upload will silently fail and the record will have null storage_path.
- No delete functionality for generation records in this ticket.

### Manual SQL

Run the migration `supabase/migrations/20260526_contract_04f_generation_history.sql` against the target Supabase database.
